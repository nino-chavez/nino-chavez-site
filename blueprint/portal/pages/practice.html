03 — Operating rules

Four rules quoted verbatim from ~/.claude/CLAUDE.md.

The global instructions file is the source of truth for how every agent on this workstation operates. These four rules carry the most cross-project leverage — they shape every session, not just specific workflows.

R1 · Decision bias

Default to action, not confirmation.

"Don't pause to ask which direction to take when the direction is obvious from the conversation. If we've been working on X and there's a natural next step on X, take it. Mid-task 'continue or pause?' questions kill flow and read as timidity, not care."

~/.claude/CLAUDE.md §"Decision bias"

R2 · Canonical-pattern-first

Vendor canonical, then internal reference, before custom shapes.

"When implementing infrastructure with a well-known canonical pattern — auth, payments, OAuth, sessions, queues, webhooks — default to the canonical pattern. Custom shapes must justify themselves explicitly against the canonical baseline."

~/.claude/CLAUDE.md §"Canonical-pattern-first for infrastructure"

R3 · Worktree isolation

Multi-session work isolation is mandatory.

"Any time more than one session has independent work in flight against the same repo, each session MUST operate in its own git worktree. Sessions sharing a working directory will switch each other's branches under each other's feet."

~/.claude/CLAUDE.md §"Multi-session work isolation"

R4 · Grounded interior state

Never fabricate. Always ground self-interrogation in evidence.

"In prose drafted on Nino's behalf, never invent specific people, conversations, internal admissions, or experiences. Self-interrogation without grounding evidence is decoration that fails the voice's central test."

~/.claude/CLAUDE.md §"Never fabricate Nino's interior state"
